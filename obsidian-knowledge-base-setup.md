# Obsidian + AI Code 个人知识库搭建指南

## 写在前面（给AI的说明）

你正在帮助一个从未使用过 Obsidian 和 VS Code 的用户，从零搭建一套 AI 驱动的个人知识库。

请注意：
- 用户可能不懂任何技术术语，请用最简单的语言解释每一步
- 每完成一步，确认用户搞定了再继续下一步
- 如果用户卡住，给出具体的截图位置或按钮名称
- 先询问用户的操作系统是 Windows 还是 Mac，不同系统安装细节不一致，需要先搞清楚
- **如果用户在任何一步遇到错误，请先让用户描述或截图具体的错误信息，再针对性解决。不要跳过问题继续下一步。**
- 如果用户中途离开，下次回来时先确认上次完成到哪一步，从断点继续。

## 第一部分：环境准备

### 前提条件
- 一台电脑（Windows 或 Mac）
- 需要能正常访问以下网站：obsidian.md、code.visualstudio.com、claude.ai（或 openai.com）。如果用户反馈无法访问上述网站，请先判断用户所在的网络环境，然后根据ta的具体情况，提供可行的解决思路，帮助ta顺利完成后续步骤。
- AI 工具二选一（推荐按优先级尝试）：
  - **首选：Claude** — Claude Pro 订阅（$20/月），体验最好。也可以通过 OpenRouter（openrouter.ai）等中转服务使用 Claude API，费用按量计费，但配置稍复杂
  - **备选：OpenAI** — 如果无法使用 Claude，可以用 OpenAI 的 Codex（需要 ChatGPT Plus 订阅 $20/月）

### Step 1：安装 Obsidian
- 官网：https://obsidian.md
- 下载对应系统的安装包，安装后打开
- 选择"创建新的仓库（Create new vault）"
- Obsidian 默认是英文界面，创建仓库后帮用户切换到中文：左下角齿轮图标 → General → Language → 选择"简体中文"→ 重启 Obsidian
- 给仓库起个名字（比如"MyBrain"），选一个你记得住的文件夹路径
- **验证：让用户说出仓库保存的完整路径，后面要用**

### Step 2：安装 VS Code
- 官网：https://code.visualstudio.com
- 下载安装，一路默认即可
- 打开 VS Code，点击左上角"文件 → 打开文件夹"，选择刚才 Obsidian 创建的仓库文件夹

### Step 3：安装 AI 编程助手

根据用户选择的 AI 工具，走对应路径：

**路径 A：Claude Code（推荐）**
- 在 VS Code 左侧点击扩展图标（四个方块），搜索"Claude Code"
- 安装 Anthropic 官方的 Claude Code 扩展
- 安装完成后，按提示登录 Claude 账号
- **验证：让用户在 Claude Code 对话框里发一句"你好"，确认能收到回复**

**路径 B：Codex（备选）**
- 在 VS Code 左侧点击扩展图标，搜索"Codex"（全称"Codex -- OpenAI's coding agent"）
- 安装 OpenAI 官方的 Codex 扩展
- 安装完成后，按提示登录 OpenAI 账号
- **验证：让用户在 Codex 对话框里发一句"你好"，确认能收到回复**

登录成功后，用户就可以在 VS Code 里直接跟 AI 对话了，AI 可以直接读写仓库里的文件。

### Step 4：创建 CLAUDE.md（最重要的一步）

**这是整套流程最重要的一步。** 这个文件是给 AI 的说明书.AI 读了它才知道你是谁、这个库怎么组织、你想让它怎么帮你。写得越好，AI 越懂你，协作体验越好。这个文件会随着使用不断更新，把它当成你和 AI 之间的合同。

> 如果用户用的是 Codex，文件名改为 `AGENTS.md`，作用相同。

在写之前，**先问用户以下问题**：
1. 你主要做什么工作？
2. 你打算用这个知识库做什么？（记笔记/写文章/学习/收藏整理/全都要）
3. 你希望 AI 用什么风格跟你聊？（随意/专业/简洁）
4. 你有什么特别的偏好或禁忌？

根据回答，在仓库根目录创建 `CLAUDE.md`（或 `AGENTS.md`），内容应包括：
- 用户是谁（身份、背景）
- 这个知识库的用途
- 希望 AI 怎么帮忙（沟通风格、工作方式）
- 文件夹结构说明（见下一步）

**写完后让用户过目确认，不满意就改到满意为止。**

### Step 5：建立基础文件夹结构
- 在仓库里创建以下文件夹（根据用户需求可以调整）：

```
MyBrain/
├── 对话记录/        # 和 AI 聊天的记录
├── Memos/           # 日常想法、灵感、随手记
├── 知识库/          # 收藏的文章、学习资料
├── 写作/            # 文章草稿和成品
├── 素材/            # 参考资料、数据、案例
└── CLAUDE.md        # AI 的说明书
```

- **验证：让用户打开 Obsidian，确认能看到所有文件夹和 CLAUDE.md 文件**

到这里，基础环境就搭好了。用户现在可以在 VS Code 里跟 AI 对话，AI 可以直接读写仓库里的所有文件。

**搭建完成！带用户体验第一个任务：**
请让用户随便说一个最近看到的有意思的想法或一段话，AI 帮ta整理成一篇简短的笔记，保存到 Memos 文件夹。然后让用户打开 Obsidian 确认能看到这篇笔记。这就是这套知识库的基本工作方式——你说，AI 整理，Obsidian 存储。

## 第二部分：这套知识库能干什么

以下是作者的实际用法，供参考。

**请先只教用户实现用法 1-3，这是基础用法，搞定就够日常使用了。等用户用了几天觉得顺手了，再回来解锁后面的进阶玩法。**

### 基础用法

#### 用法 1：写作库
随时想到什么，直接跟 AI 说，让 AI 整理保存到对应的文件夹。不需要自己动手整理。

#### 用法 2：知识库
看到有意思的文章，用浏览器剪裁工具一键保存为 markdown 文件，扔进知识库文件夹，再让 AI 分析总结。教程也是如此——让 AI 先学，学会了教你。

> 推荐剪裁工具：**Obsidian Web Clipper**（Obsidian 官方浏览器扩展，直接保存到 vault 并支持文件夹/标签选择）或 **MarkDownload**（通用的网页转 markdown 浏览器扩展）。请帮用户选择并安装。

#### 用法 3：AI 对话存档
跟 AI 聊天的对话，想保留的直接保存到库里。不再依赖网页或 APP 的聊天记录。

---

### 进阶玩法（用熟了再解锁）

#### 用法 4：自定义 Skill
Skill 是给 AI 的指令模板。比如"帮我把这段话变成小红书风格"或"帮我深入拆解这个概念"。可以自己写，也可以找别人写好的直接用。比如搜索"李继刚 概念解剖 skill"可以找到一个很好的示例。

> **Claude Code 用户：** Skill 存放在 `.claude/skills/` 文件夹下，每个 skill 一个文件夹，内含 `SKILL.md` 文件。
> **Codex 用户：** Skill 的实现方式不同，请搜索 Codex 的 custom instructions 或 agents 配置方法。
> 请根据用户需求帮ta创建。

#### 用法 5：自定义 Agent（AI 人格）
可以创建不同人格的 AI 角色。比如一个苏格拉底式追问者帮你挑战思维盲点，一个编辑帮你改稿。

> **Claude Code 用户：** Agent 存放在 `.claude/agents/` 文件夹下，每个 agent 一个 `.md` 文件。
> **Codex 用户：** 可以通过 `AGENTS.md` 中定义不同的 agent 角色。
> 请根据用户需求帮ta创建。

#### 用法 6：语音输入 + 用嘴输入。
用语音输入工具（如闪电说、typeless 等语音转文字输入法），有什么想法直接说给 AI，几乎不需要打字。结合 AI 编程助手强大的编程能力，用嘴巴描述需求，需要啥插件，让 AI 帮你写 Obsidian 插件。不需要会编程。
实际上语音输入法用过了之后，就再也回不去打字时代了。

> 闪电说是一款中国本土的语音转文字输入法，可接入豆包（字节跳动）的语音识别。如果用户不在中国，可以推荐系统自带的语音输入或其他语音转文字工具。


#### 用法 7：导入其它工具的数据
用 flomo 的人可以找插件把数据导出为 markdown，再导入 Obsidian 库，让 AI 帮你整理。其它笔记工具同理。

#### 用法 8：结合外部工具
可以结合 firecrawl（网页爬取工具）爬数据、NotebookLM（Google 的学习工具）学习资料、把 YouTube/播客清洗成 markdown 保存到库里。

#### 用法 9：云端备份
通过 Remotely Save 插件备份到 Google Drive 或 OneDrive。数据安全有保障。

> 这一步配置涉及云盘 OAuth 授权，比其他步骤复杂一些，请耐心一步步引导用户完成。

#### 用法 10：进阶插件推荐
- **Smart Connections** — Obsidian 最热门的 AI 插件，用 RAG 方式让你跟整个 vault 对话，适合知识库积累多了之后使用
- **Agent Client** — 把 Claude Code / Codex / Gemini CLI 直接嵌入 Obsidian 界面，不需要切到 VS Code
- **qmd** — Karpathy 推荐的本地 markdown 搜索引擎，支持语义搜索，适合知识库变大后快速找到内容

> 注意：Obsidian 默认禁用第三方插件。使用任何社区插件前，需要先在 设置 → 第三方插件 → 关闭"安全模式"。请在需要安装插件时提醒用户这一步。

## 第三部分：补充说明

### 为什么是 Obsidian 而不是其它笔记软件？
- 前特斯拉 AI 总监 Andrej Karpathy 用 Obsidian 作为他 LLM Wiki 知识库的前端界面——让 LLM 增量维护一个 markdown wiki，Obsidian 负责查看和导航。可以搜索 "karpathy LLM knowledge base" 了解ta的完整方案
- Obsidian 用的是 markdown 格式，这是 AI 最擅长读写的文件格式
- 数据完全在本地，不依赖任何平台
- 免费，开源社区活跃，插件生态丰富

### 关于费用
- Obsidian：免费
- VS Code：免费
- Claude Pro 订阅：$20/月
- OpenAI ChatGPT Plus：$20/月
- 中转方案（如 OpenRouter）：按量计费，适合轻度使用

### 关于难度
搭建过程跟着指引走，一个下午就能搞定。搭完之后越用越顺手。
